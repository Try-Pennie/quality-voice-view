## Design Context

### Users
**Eavesly is an internal Pennie tool — not consumer-facing.** Two audiences share the same surface:

1. **Sales managers (primary)** — review Slack-flagged calls and capture structured feedback. Context: triaging between meetings, reading on a laptop, want to clear the queue and coach agents fast. They open the dashboard from a Slack alert deep-link or via the calls list.
2. **Executives (secondary, less frequent)** — glance at trends and team-level rollups. Context: weekly check-ins, want signal not noise.

**Job to be done**: Make manager review of flagged calls fast and structured enough that the resulting feedback can be used to improve the upstream QA model. Speed and trust matter more than discovery — managers know what they need to do; the UI's job is to get out of their way.

### Brand Personality
Eavesly inherits the **Pennie brand** (see `pennie-brand` skill). Three words: **empowering, accessible, human**. The dashboard is the operator-facing side of a brand whose consumer voice is "your personal finance partner" — so the internal tool should feel like the same company, just turned inward: warm, plain-spoken, considered. Not surveillance-y. Not cold fintech. Not a "compliance hammer." It's a coaching tool.

Emotional goal: managers should feel **calm and in control** when reviewing alerts, not buried in noise.

### Aesthetic Direction
**Pennie brand system, applied to a dense internal dashboard.** Reference apps: Linear (clarity, generous whitespace), Stripe Dashboard (data density without clutter), Pennie's own marketing site (warm beige neutrals, soft pastel cards, navy text). Theme: **light only** (no dark mode). The interface is type-driven, restrained color, blue as the default accent.

**Anti-references** (must NOT look like):
- Generic shadcn-default dashboard (gray neutrals, sharp corners, drop shadows on everything)
- AI-slop dashboard (cyan-on-dark, purple-blue gradients, neon accents, glassmorphism, hero-metric layouts, identical card grids)
- Surveillance/police-state UI (red everywhere, "violations" framed as accusations)

**Locked design tokens (from Pennie brand):**
- **Type**: PP Mori primary (Medium 500 minimum — never Regular 400). Helvetica Now Display reserved for hero H1 only. Fallback stack: `'PPMori', 'Inter', system-ui, sans-serif`. Small labels: uppercase, Bold 700, +0.6–1px letter-spacing.
- **Neutrals**: White `#FFFFFF`, Beige `#F9F6F0`, Graphite `#343434` (body text), Navy `#1D212F` (headings, dark surfaces). **Never pure black.**
- **Accents** (sparingly): Blue is the default — Light `#ECF8FF`, Main `#82D0FF`, Dark `#439FE1`. Other families (Green `#86E7B3`/`#55CF90`, Yellow `#FFD073`/`#D69938`, Peach `#FF966F`/`#E7704C`, Indigo `#73A2FF`/`#486CCA`) only when encoding distinct meaning. Never accent-on-accent.
- **Radius**: pill `100px` (buttons, badges, chips), card `20–24px`, small chip `8–12px`, icon container `14–16px`. **Never 4px or 6px.**
- **Shadows**: Navy-based — `0 2px 12px rgba(29,33,47,0.07)` resting, `0 8px 32px rgba(29,33,47,0.10)` floating, `0 16px 48px rgba(29,33,47,0.14)` elevated. Never pure black.
- **Spacing scale**: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 60 / 80 px. Card internal padding minimum 24px, prefer 32–40px. Generous whitespace is a brand signal.
- **Badges**: pill, light-accent bg, dark-accent text, with a 7px filled dot preceding the label, uppercase Bold 700, +0.6px tracking.
- **Icons**: never bare — sit in 48–56px rounded-square containers (radius 14–16px) with light-accent backgrounds.
- **Motion**: entry `translateY(20–30px) → 0` + opacity 0→1, 0.5–0.7s, `cubic-bezier(0.16, 1, 0.3, 1)`, 0.1s child stagger. Hover on cards `translateY(-3px)` 0.2s. **No bounce, no elastic, no rotation.**

### Design Principles

1. **The queue is the product.** Optimize for managers clearing alerts fast. Every interaction in the review flow (open → judge accuracy → tag → submit → next) must be reachable by keyboard and feel like one motion. Decoration that slows this down loses.

2. **Pennie brand, not shadcn defaults.** Replace the generic HSL gray palette with the Pennie token system. Beige and Navy are real surfaces here — not exotic. Blue is the default accent. Other colors only when they encode meaning (e.g., severity, status).

3. **Show structure, hide noise.** Five violation types and ~20 modules across the system — the UI must make the *category* legible at a glance (badges, chips) without burying the call-level detail. No card-grids of identical tiles. Use rhythm: tight groupings for related metadata, generous separation between zones.

4. **Restraint over volume.** Whitespace, type weight, and one accent color do most of the work. Reserve red/peach for actual violations, green for confirmed-accurate reviews, blue as everything-else default. Never gradient text, never glassmorphism, never hero-metric hero-numbers.

5. **Plain-spoken microcopy.** Match Pennie's consumer voice: warm, direct, no jargon. "Was this alert accurate?" not "Adjudicate flag validity." "Coached agent" not "Remediation action recorded." Empty states should teach, not just say "nothing here."
