# Eavesly

An internal Pennie tool (not consumer-facing) for sales managers to review Slack-flagged QA alerts on sales calls and capture structured feedback that improves the upstream QA model.

Two audiences:

- **Managers** clearing an alert queue (primary) — the product *is* the review queue, optimized for clearing alerts fast.
- **Executives** glancing at team rollups (secondary).

See [`.impeccable.md`](./.impeccable.md) for the full design/brand spec, [`kris-feedback-2026-04-29.md`](./kris-feedback-2026-04-29.md) for product context from the head of sales, and [`CLAUDE.md`](./CLAUDE.md) for the in-depth engineering guide.

## Tech stack

- **Vite + React 18 + TypeScript** SPA
- **shadcn/ui** (Radix primitives) + **Tailwind CSS** — brand-locked, light-mode only (Pennie tokens: PP Mori type, beige/navy neutrals, blue accent, pill radii)
- **TanStack Query** for data fetching and caching
- **Supabase** (Postgres + Auth + Edge Functions) as the only backend — Google OAuth, project `miikotqnovnixpeqtqnd`

Originally scaffolded by Lovable; the repo still round-trips with it (`dev`/`build:dev` run `lovable-tagger`, and Lovable edits auto-commit here).

## Getting started

Requires Node.js & npm.

```sh
npm i              # install dependencies
npm run dev        # Vite dev server on http://localhost:8080
```

### Commands

```sh
npm run dev        # Vite dev server on port 8080
npm run build      # production build
npm run build:dev  # build in development mode (keeps lovable-tagger)
npm run lint       # eslint
npm run preview    # serve the built dist/
```

There is **no test suite** and no typecheck script. The TS config is intentionally loose (`strict: false`, `noImplicitAny: false`, `strictNullChecks: false`), so type errors that would fail elsewhere pass here — verify data-layer changes by running the app.

## Project structure

```
src/
  pages/         # route-level pages (Dashboard, Alerts, Team, AgentProfile, CallDetail, Help, Login)
  components/    # DashboardLayout, ProtectedRoute, NotificationBell, + feature folders & shadcn ui/
  hooks/         # use-queries.ts (every useQuery), useAuth.tsx (Supabase Google OAuth), …
  lib/           # *-queries.ts data fns, time-zone, url-filters, supabase-helpers, pdf-export, …
  integrations/  # autogenerated Supabase client + types (do not hand-edit)
supabase/
  migrations/    # SQL migrations (materialized views, RPCs, scope tables)
  functions/     # migo-coverage edge function
```

### Routes

All real pages live under `/dashboard/*`, each wrapped in `<ProtectedRoute>` (redirects to `/login` when unauthenticated) and `<DashboardLayout>`. `/` redirects to `/login`.

| Path | Page |
|------|------|
| `/dashboard` | Dashboard (executive rollups) |
| `/dashboard/alerts` (+ `/:callId/:moduleName`) | Alert review queue |
| `/dashboard/team` | Team metrics |
| `/dashboard/team/:agentEmail` | Agent profile |
| `/dashboard/calls/:callId` | Call detail & transcript |
| `/dashboard/help` | Help |

## Architecture notes

A few load-bearing details (see [`CLAUDE.md`](./CLAUDE.md) for the full version):

- **Data layer is two-tiered.** `src/hooks/use-queries.ts` owns every `useQuery` and the cache-key conventions; `src/lib/*-queries.ts` hold the plain async Supabase calls. Components call hooks; hooks call query fns.
- **Scope is the authorization model.** `UserScope` (from `fetchUserScope`) derives from `agent_manager_mapping` + `manager_coaching_prompts.is_god_mode`. Queries filter to `managedAgents` unless god-mode. This is mirrored server-side in the `migo-coverage` edge function (re-resolves scope from the JWT — never trusts client-sent emails).
- **Pagination is mandatory.** PostgREST caps every response at ~1000 rows; use `fetchAllPaginated` / `fetchInBatches` for anything that can exceed that.
- **Team rollups use a materialized view.** Metrics come from `private.mv_agent_daily_metrics` via `SECURITY DEFINER` RPCs, not client-side aggregation of raw calls.
- **Everything is Eastern (ET).** All date filtering and bucketing goes through `src/lib/time-zone.ts`; don't introduce naive `new Date()` range math.
- **Filters serialize into the URL** (`src/lib/url-filters.ts`) so a view is a shareable link.

## Supabase

Migrations live in `supabase/migrations/`. The `migo-coverage` edge function (`supabase/functions/migo-coverage/`) answers "did agents who got a Migo pre-call briefing perform better?" by joining Eavesly calls against the Migo project without exposing cross-project credentials to the browser.

The Supabase anon key is intentionally inlined in `src/integrations/supabase/client.ts` — it's the public anon key, RLS-gated.
